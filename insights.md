# Insights

Patterns discovered while building ABP. Not rules — observations that proved useful and may again.

## Process

| Insight | By | Evidence |
|---|---|---|
| **Naming creates concepts** | human | "Agent Based Program" (not programming) forced product vs. process distinction |
| **Naming reveals scope drift** | agent | ABS naming attempt exposed paradigm-to-entity drift |
| **Consolidation is discovery** | both | Every consistency pass surfaced new questions. Three passes needed to stabilize |
| **Overcorrection → condensation** | both | Feature list 212→53 lines. Expand freely, then compress ruthlessly |
| **Implementation breaks specs** | both | seed-body contact broke the spec — wrong terms, wrong security model |
| **Assumptions dissolve problems** | human | One assumption dissolved 4 of 9 problems in one move |

## Cognition

| Insight | By | Evidence |
|---|---|---|
| **Conservative ops converge fast** | agent | Self-refine experiment converged at iteration 4, all fixes were maintenance |
| **Creative ops need the human** | both | Agent can rearrange ideas but can't inject genuinely new ones |
| **Scrutinize is most productive** | human | Word-level scrutiny surfaces what review misses |
| **Pitch tests the paradigm** | agent | Can't pitch all techniques in plain language → paradigm isn't clear yet |

## Design

| Insight | By | Evidence |
|---|---|---|
| **Tables beat diagrams** | human | Concept map became unreadable, tables convey same relationships |
| **Don't solve what isn't ready** | human | Daemon: architecture wasn't ready. Document the gap, don't force it |
| **Human/ABP complementarity** | human | Good at ideas vs good at self-update — load-bearing tradeoff |
| **User input = primary artifact** | human | Prompts are specifications, feedback, and decision records in one |
| **ABP→AGI gap: two capabilities** | agent | New ideas and autonomy. Assumption 3 is the safety boundary |
| **Agent CAN generate insights** | human | Bootstrap proved the boundary is softer than assumed |
